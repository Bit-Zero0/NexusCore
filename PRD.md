
# 二、NexusCode 总需求文档

## 1. 产品概述

### 1.1 产品名称
NexusCode

### 1.2 产品目标
NexusCode 是一个统一开发者平台，支持：
- 编程题评测
- 内容创作与发布
- 云函数执行
- 双链笔记管理
- 实时消息与执行反馈

### 1.3 目标用户
- OJ 用户与出题人
- 开发者与算法学习者
- 技术博客作者
- 使用双链笔记进行知识管理的用户
- 需要在线函数执行能力的开发者

---

## 2. 核心业务域需求

## 2.1 OJ 系统需求

### 2.1.1 题目管理
系统应支持：
- 新建题目
- 编辑题目
- 删除题目
- 发布/下线题目
- 查看题目详情
- 批量导入题目
- 多题导入 JSON
- 题目标签、难度、题型管理

### 2.1.2 判题模式
系统应支持：
- ACM 判题
- Functional 判题
- EasyJudge 判题
- Validator 判题
- SPJ 判题

### 2.1.3 测试用例管理
系统应支持：
- JSON 方式录入测试用例
- 表单方式逐组录入测试用例
- 两种模式互相转换
- 测试用例校验
- 限制测试用例数量与结构合法性

### 2.1.4 资源限制
系统应支持：
- 按语言设置时间限制
- 按语言设置空间限制
- 时间单位：`ms / s`
- 空间单位：`KB / MB`
- 默认值策略：
  - C++ 默认 `1s / 50MB`
  - Python 默认 `2s / 100MB`

### 2.1.5 提交与结果
系统应支持：
- 提交代码或答案
- 提交后生成唯一提交记录
- 实时订阅判题状态
- 查看最终结果
- 查看错误信息
- 查看测试点明细
- 查看提交代码

### 2.1.6 简单题需求
EasyJudge 应支持：
- 判断题
- 单选题
- 多选题
- 动态选项配置
- 选项描述配置
- 标准答案支持：
  - `"AB"`
  - `["A", "B"]`

### 2.1.7 集群与调度
系统应支持：
- 节点心跳与状态上报
- 队列路由
- `fast / normal / heavy` 队列分层
- 自动重试
- 死信队列
- MQ 重连
- 实时集群监控

---

## 2.2 内容与博客系统需求

### 2.2.1 文档编辑与存储
系统应支持：
- Markdown 内容编辑
- 平台扩展语法
- 文档草稿
- 文档保存
- 文档版本管理
- 元数据配置

### 2.2.2 双链能力
系统应支持：
- `[[双链引用]]`
- 反向链接
- 文档引用图谱
- 块级引用
- 文档之间的知识关系展示

### 2.2.3 发布能力
系统应支持：
- 将文档发布为博客文章
- 管理发布状态
- 控制是否公开
- 更新已发布内容
- 未来支持多发布目标

### 2.2.4 渲染能力
系统应支持：
- 标准 Markdown 渲染
- 平台扩展语法渲染
- 代码块高亮
- 自定义分栏渲染
- 局部高亮
- 引用增强
- 可扩展 AST 渲染

---

## 2.3 云函数需求

### 2.3.1 函数管理
系统应支持：
- 创建函数
- 编辑函数
- 发布函数
- 查看函数版本
- 删除函数

### 2.3.2 函数执行
系统应支持：
- 手动调用
- 异步调用
- 资源限制
- 执行日志
- 执行结果
- 执行失败回退与重试

### 2.3.3 运行环境
系统应支持：
- 多语言函数
- 沙箱执行
- 超时限制
- 内存限制
- 安全隔离

---

## 2.4 笔记客户端需求

### 2.4.1 基础能力
桌面端应支持：
- 本地文档管理
- 文件夹结构
- 双链
- 标签
- 搜索
- 预览
- 分栏阅读

### 2.4.2 扩展语法
客户端应支持平台语法：
- 分栏
- 代码块区域高亮
- 自定义块语法
- 未来更多扩展节点

### 2.4.3 发布能力
客户端应支持：
- 登录平台
- 发布文档
- 更新已发布文章
- 同步平台内容
- 管理发布记录

---

## 3. 非功能需求

### 3.1 性能需求
- 支持高并发 HTTP 请求
- 支持大量 WebSocket 连接
- 支持异步任务调度
- 支持实时判题状态推送

### 3.2 安全需求
- 平台鉴权
- 权限控制
- 资源隔离
- 不可信代码执行隔离
- 后续支持审计日志

### 3.3 可维护性需求
- 明确模块边界
- 可观测性
- 配置分层
- 文档完整
- 支持后续扩展

### 3.4 可扩展性需求
- 可扩展业务域
- 可扩展内容语法
- 可扩展执行模型
- 可拆分部署
